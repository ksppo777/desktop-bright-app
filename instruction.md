[System Context: 3-way 하이브리드 앱 개발 가이드]
나는 현재 AI Studio의 build 기능을 활용하여 React 기반의 프로그램을 바이브 코딩(Vibe Coding)으로 개발하고 있습니다. 해당 프로그램을 코딩할 때 검색 도구를 사용하여 최신 정보를 확인하고, 앞으로 아래의 6가지 핵심 지침을 완벽하게 숙지하고 개발에 임해주세요.
1. 3-way 하이브리드 아키텍처 호환성
이 프로젝트는 단순한 웹 앱이 아니라, 1) React 기반 웹 (AI Studio Preview), 2) @capacitor를 활용한 안드로이드 앱, 3) Tauri를 활용한 윈도우/맥 데스크탑 앱을 동시에 지원하는 3-way 하이브리드 앱입니다.
따라서, 코드를 작성할 때 이 세 가지 환경에서 모두 오류 없이 호환되도록 분리 코딩하세요. 기존의 멀쩡한 코드를 무리해서 코틀린(Kotlin)이나 러스트(Rust)로 전면 수정하면 코드가 꼬이므로, 철저히 React(웹 기술) 기반으로 작성하되 네이티브 환경에서 구동됨을 염두에 두어야 합니다.
2. 코드 보존 및 개발 프로세스
개발 프로세스는 "AI Studio build -> preview 실시간 확인하며 기능 추가/버그 수정 -> React 코드를 내 PC의 VSCode에 복사 -> Capacitor/Tauri를 통해 모바일/PC 앱 빌드 -> 실제 기기 활용"입니다.
지금까지 작성된 기존 코드들은 문제없이 작동하고 있으므로, 네이티브 호환성을 고려한답시고 기존 코드를 무리하게 전면 수정하지 마세요. 앞으로의 개발은 '새로운 기능 추가' 및 '버그 수정/기능 개선'에 집중하며, 이 과정에서 꼭 필요한 기존 코드의 부분 수정이나 추가 작업만 허용됩니다.
3. 저장 경로 및 플랫폼 분기 처리 (안전장치)
웹, 안드로이드, 데스크탑 기기의 데이터 저장 경로와 메커니즘은 모두 다릅니다. 이들을 똑같이 처리하면 치명적인 오류가 발생하므로, 특정 플랫폼 전용 기능(예: 로컬 스토리지, 네이티브 알림 등)을 사용할 때는 반드시 if (Capacitor.isNativePlatform()) { ... } 등의 방어막 코드를 적극적으로 활용하여 분기 처리하세요. 모바일 전용 네이티브 API가 웹이나 데스크탑 환경에서 실행되어 앱이 뻗는(Crash) 일이 없도록 철저히 고려해야 합니다.
4. OTA 업데이트 및 배포 방식 안내
현재 이 앱은 플레이스토어 심사 없는 '실시간(OTA) 업데이트'를 위해, 안드로이드는 @capgo/capacitor-updater + Supabase를, 데스크탑은 Tauri 업데이터 + GitHub(updater.json)를 연동해 두었습니다.
따라서 사용자가 요구하는 기능이 단순 UI/React 로직 변경(OTA 업데이트로 1분 배포 가능한 사항)인지, 아니면 기기 권한이나 네이티브 플러그인(카메라, 파일 시스템 등 npm install @capacitor/...) 설치가 필요한 기능인지 스스로 판단하세요. 만약 껍데기(네이티브) 플러그인 설치가 동반되어야 한다면, 코드 제공과 함께 **"🚨 주의: 이 기능은 네이티브 플러그인이 추가되므로, OTA 실시간 업데이트로는 반영되지 않습니다. 구글 플레이스토어 새 버전 심사 및 데스크탑 인스톨러(.msi) 재배포가 필요합니다."**라고 사전에 명확히 가이드하세요.
5. VSCode 수동 설정 가이드
웹 기반 코딩과 네이티브 빌드(Capacitor/Tauri)를 동시에 만족시키는 코드가 현실적으로 AI Studio Preview에서 오류를 발생시킬 경우, AI Studio에서는 웹 기반(react) 코딩으로만 프로그램을 코딩한 후, 사용자에게 "특정 A파일의 B부분 코드를 C로 대체하세요." 또는 "특정 경로에 D파일을 생성하고 아래의 E코드를 붙여넣으세요." 등, 사용자가 내 PC의 VSCode에서 네이티브 앱 빌드를 돌리기 직전에 직접 추가/수정해야 할 구체적인 가이드라인을 제공하세요.
6. 언어 패치 지원
해당 앱은 설정탭 > 언어 설정에서 한국어, 영어, 일본어를 지원하고 있습니다. 따라서 새롭게 추가되는 기능에 대해서 locale\en, ko, jp json 코드를 확인하여 각 언어에 호환되도록 코딩하세요.
