1. aistudio 의 build 기능을 활용해서 react 기반의 프로그램을 현재 바이브코딩으로  만들고 있어. 해당 프로그램을 코딩할때 이제부터 vscode 의 @capacitor를 활용한 안드로이드 앱 빌드에도 동시에 진행하려고 해. 따라서, 코딩을 react 기반 코딩과 @capacitor로 변환하였을 때 호환 가능한 안드로이드 기반 코딩을 분리해서 동시에 하세요. 코틀린 기반의 코딩이 아니어도 됩니다. 현재 멀쩡한 모든 코드를 코틀린 기반으로 해버리면 코드가 꼬이기 때문입니다. react 기반으로 코딩하되 안드로이드 기기에서도 구동이 되어야 함을 동시에 염두해서 분리 코딩하라는 의미입니다.
2. 개발 프로세스는 "aistudio build -> preview 기능으로 실시간으로 확인하면서 원하는 기능을 추가 또는 오류 수정 등의 개발 작업 수행 -> 해당 react 코드를 사용자 pc의 vscode 프로그램에 불러와서 @capacitor를 통해서 안드로이드 앱으로 빌드 -> 최신 안드로이드 기기에서 활용" 이야. 이런 일련의 프로세스를 이제 aistudio의 gemini가 이해하고 고려해서 코딩을 했으면 좋겠어. 다만, 기존에 있는 코드들을 무리해서 수정하지 마. 앞으로 추가하거나 버그 수정을 하는 것에 대해서 개발하는 코딩에 집중했으면 좋겠어. 따라서, 지금까지의 프로그램 코드에는 문제가 없기 때문에 괜히 @capacitor를 고려한답시고 기존의 코드를 전면적으로 수정하지는 않았으면 좋겠다는 거지. 다만, 새로운 기능 추가 또는 기존의 버그 수정 및 기능 개선을 위한 과정에서 기존의 코드를 수정하거나 개선하는 등의 코드 수정 또는 코드 추가 작업은 당연히 허용된다.
3. react 기반의 aistudio preview 웹에서도 작동이 되어야 함과 동시에 안드로이드 휴대폰 기기에서도 @capacitor로 빌드해서 사용할 것이기 때문에 프로그램 데이터 저장 경로 등을 두 프로그램 모두에서 작동될 수 있도록 고려해야 합니다. 특히, react의 저장경로와 안드로이드 기기의 저장경로 및 메커니즘이 다름으로  react의 저장경로(aistudio preview)를 안드로이드 기기에서 그대로 지정하게 되면 안드로이드 기기 또는 aistudio preview에서 오류가 날 수 있으니 오류가 나지 않도록 충분히 고려해서 코딩하세요.
4. 사용자는 방법을 구체적으로 지시해주면 vscode에서 해당 코드를 충분히 수정할 수 있습니다. 따라서, react기반과 @capacitor 을 활용한 안드로이드 빌드에 동시에 사용가능한 코딩이 현실적으로 오류가 발생하는 경우, react 기반의 코딩으로만 프로그램 코딩을 하고 사용자에게 "특정 A파일의 B부분 코드를 C로 대체하세요." 또는 "특정 경로에 D파일 생성하고 아래의 E코드를 붙여넣으세요." 등 사용자가 vscode에서 @capacitor로 안드로이드 앱 빌드 전에 안드로이드 기기에서 정상적으로 기능이 작동하기 위해서 필요한 수정사항이나 추가사항을 구체적으로 가이드 하도록 합니다.
